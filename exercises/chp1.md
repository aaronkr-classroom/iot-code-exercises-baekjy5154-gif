# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
  센서와 액추에이터는 디바이스에서 서로 반대되는 역할을 수행한다. 센서는 주변 환경의 정보를 감지하여 디바이스로 전달하는 입력 장치로 온도나 빛, 소리 등을 측정하는 기능을 한다. 반면 액추에이터는 디바이스로부터 받은 신호를 바탕으로 실제로 움직이거나 작동하는 출력 장치로 모터를 회전시키거나 소리를 내는 등의 동작을 수행한다.

  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
  센서는 주변 정보를 감지해 입력하는 장치이고 액추에이터는 명령을 받아 실제로 동작하는 출력 장치이다. 또한 디바이스와 대시보드는 환경과 목적이 달라 각각에 적합한 프로그래밍 언어를 사용하기 때문에 동일한 언어를 쓰지 않는다.

  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
브로커와 같은 정보 전달 전담 서버는 디바이스와 대시보드 사이에서 데이터를 대신 주고받아 주는 역할을 한다. 이를 통해 서로 직접 연결하지 않아도 되어 구조가 단순해지고, 여러 기기가 동시에 안정적으로 통신할 수 있으며, 데이터 관리와 확장이 쉬워지는 장점이 있다.

2. Q2
  - (a) Mosquitto와 Node-RED를 별도 서버에 설치할 경우의 이점을 설명하세요.
별도 서버에 설치하면 각 시스템이 독립적으로 동작하여 성능 저하나 장애가 서로에게 영향을 주지 않는다. 또한 역할이 분리되어 유지보수와 확장이 쉬워지고 필요에 따라 각각의 서버 성능을 따로 조정할 수 있어 전체 시스템의 안정성과 효율성이 높아진다.

  - (b) Mosquitto와 Node-RED를 운영하기 위한 적절한 컴퓨팅 환경을 기술하세요.
Mosquitto와 Node-RED를 운영하기 위해서는 안정적인 서버 환경이 필요하다. 일반적으로 리눅스 기반의 운영체제에서 실행하는 것이 효율적이며 네트워크 연결이 안정적이어야 한다. 또한 동시에 여러 디바이스가 접속할 수 있으므로 적절한 CPU와 메모리 지원을 갖추는 것이 중요하고 저장공간과 보안 설정도 함께 고려해야 한다.

4. 스마트 홈 구축을 위한 아이디어를 다음 조건을 준수하여 정리해보세요.

  - 사용할 센서 목록 작성
  - 대시보드 구성 항목 정리
  - 시스템 통제 시나리오 작성
